# 简介

ClawCenter 是一个中央路由器，连接你的微信和多个 AI Agent。你可以在微信聊天里直接跟 AI 编程助手（Claude Code、OpenCode 等）对话——用 `#话题标签` 在同一个聊天窗口里切换不同的 Agent。

## 要解决的问题

微信 Bot 每个账号只有一个聊天入口。如果你想用多个 AI Agent，所有消息都走同一个对话——很快就会变得混乱。你分不清在跟谁说话，也没办法维护独立的对话上下文。

## ClawCenter 的解决方式

- **#话题标签路由**：输入 `#claude 修复这个bug` 就能把消息发给指定 Agent
- **引用回复接续**：引用 Agent 的回复继续对话——不用重新打标签
- **粘性路由**：不指定 Agent 的话，消息会发给你上次聊过的那个
- **Agent 回复前缀**：每条回复都带 Agent 名称前缀（如 `[🤖 Claude]`），让你随时知道谁在说话

## 核心概念

| 概念 | 说明 |
|------|------|
| **微信账号** | 通过 iLink Bot API 连接的微信机器人 |
| **Agent** | 一个 AI 后端实例——可以是本地 CLI 进程或远程服务 |
| **会话** | 一个用户和一个 Agent 之间的独立对话 |
| **路由** | 决定哪个 Agent 处理消息的逻辑 |
| **Worker** | 运行 Agent 的远程机器，连接到中央服务器 |

## 架构概览

```
微信用户
  │
  ▼
┌─────────────────────────────────┐
│       ClawCenter（中央节点）       │
│                                 │
│  微信连接器 ──► 路由器 ──► Agent   │
│  （多个）        │      管理器     │
│                 │     （多个）     │
│              SQLite              │
│                 │                │
│   Web 管理面板 ◄─┤──► Worker Hub  │
│    :9800        │      :9801     │
└─────────────────┼───────────────┘
                  │
        ┌─────────┼─────────┐
        │                   │
     Worker A            Worker B
     （远程）              （远程）
     agents...           agents...
```

**Center 模式**：连接微信、路由消息、管理本地 Agent、接受远程 Worker。

**Worker 模式**：管理本地 Agent，连接到 Center 使它们远程可用。
